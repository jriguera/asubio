Redistribution on other web sites, CD-ROM etc

You are welcome to redistribute the GIMP on other web sites, CD-ROM, and other media. You don't have to ask for permission. That's one of the points of Free Software. One important thing that the GNU license requires is that you must also redistribute the source code. This means the GIMP, GLib, GTK+ 1.3.0 and gettext sources from this site (ftp.gtk.org is the same site as this). Also the libiconv sources at the sourceforge.net site linked to from below must be redistributed.

Installer

There is a GIMP installer that contains the appropriate stuff from these zipfiles (or relatevely fresh previous versions) at http://www2.arnes.si/~sopjsimo/gimp/. That installer is created by Jernej Simoncic, so please ask him about problems with it.

Prebuilt binary packages, developer packages, and source

These packages are for experienced GIMP and Windows users, and for people who develop software that use GLib and/or GTK. The binaries (DLLs and executables) in these packages are not for the Cygwin environment. They use the MSVCRT runtime. The files included in these zip archives are not compatible with, or drop-in replacements for, the stuff used in the GIMP for Windows from December 2000.

(For Cygwin builds of GTK etc, check out Masahiro Sakai's site.)

There are runtime and developer packages. The runtime packages include auxiliary executables, DLLs, and files needed at runtime. The developer packages include package-related developer tool executables, import libraries, headers, pkg-config files, and documentation.

(In some cases the runtime and developer packages are combined.)

For everything to work as smoothly as possibly, unzip the files below (or just those you need) into some directory of your choice. Let's call that directory top. The add top\bin and top\lib to your PATH.

pkg-config is a replacement for the glib-config and gtk-config scripts that previously were used on Unix. (No such scripts were included in my Windows port, though.)

You can use the pkg-config command in your makefiles like this:

GLIB_CFLAGS=`pkg-config --cflags glib-2.0`
GLIB_LIBS=`pkg-config --libs glib-2.0`
...
foobar.exe : $(FOOBAR_OBJS)
	   $(CC) -o $@ $(FOOBAR_OBJS) $(GLIB_LIBS)

Etc. That only works with a Unixish Make and especially with a command interpreter (shell) that understands backquotes. (I hope you notice those backquotes in the above Makefile snippet, and understand what they mean. If you don't, educate yourself.)

I advise not to use a directory with spaces or other funny character in the name as top. Otherwise you will get problems when using pkg-config output in makefiles.

If you are a MSVC user and use nmake, it's a bit more complicated. nmake (or the standard Windows command interpreter it uses, command.com or cmd.exe) doesn't support backquotes. You must run pkg-config manually with the switch --msvc-syntax, and paste its output into your nmake makefile.

On NT/2k/XP, it is possible to have nmake create temporary files containing the output from pkg-config, and use those in a nmake makefile like this: 

foobar.exe: foobar.obj __gtk_libs.mak
	cl -MD foobar.obj @__gtk_libs.mak
	@del __gtk_libs.mak

foobar.obj: foobar.c __gtk_cflags.mak
	cl -MD -c @__gtk_cflags.mak foobar.c
	@del __gtk_cflags.mak

__gtk_cflags.mak:
	pkg-config --msvc-syntax --cflags gtk+-2.0 >$@

__gtk_libs.mak:
	for /F "delims==" %i in ('pkg-config --msvc-syntax --libs gtk+-2.0') \
		do echo /link %i >$@

Note in the above makefile fragment that the __gtk_libs.mak file is created using the for /F syntax available only in the cmd.exe command interpreter on NT/2k/XP. This obscure acrobatics is needed because we want __gtk_libs.mak to contain a line that starts with /link, but pkg-config cannot output the /link flag itself as a cl command line might contain several invokations of pkg-config --libs. We cannot put the /link on the cl command line that links foobar.exe either, as cl then gets confused and runs the linker with a command file that on one line has @__gtk_libs.mak, and link.exe doesn't like that. Sigh.

  • GNU libiconv for Win32. GNU Libiconv is an implementation of iconv. GLib uses libiconv, so you will need this. To install, put the DLLs and LIBs in the top\lib directory, and put the EXE in the top\bin directory. If you intend to develop software using gcc and libtool that would use libiconv, copy iconv.lib to libiconv.a. (libtool doesn't recognize .lib files, even if GNU ld does.)

  • libintl-0.10.40-tml-20020904.zip, containing GNU libintl (from GNU gettext0.10.40) modified a bit by me for Windows. GLib uses libintl, so you will need this.

  • gettext-dev-0.10.40-20020904.zip, containing the corresponding headers, import libraries, manual pages, and the gettext commands (msgfmt.exe etc). You will need this if you rebuild glib or other stuff that uses libintl. The msgfmt command is needed if you want to recompile message catalogs. The other commands can be used to mangle message catalogs in other ways. Using some other distribution of gettext would be fine, too. Only the libintl DLL has some changes by me.

  • The sources are in gettext-0.10.40-tml-20020904-src.zip.

  • glib-2.2.1.zip Runtime package for GLib 2.2.1. Requires libiconv and libintl.

  • glib-dev-2.2.1.zip, corresponding developer package, i.e. headers, import libraries and documentation.

  • glib-2.2.1.tar.gz, corresponding sources.

  • pkgconfig-0.14.zip. Pkg-config is a neat program that is useful in makefiles etc. It manages compile and link flags. This is pkg-config 0.14. Requires GLib 2.0

  • Sources for pkg-config (On this site, to satisfy strict interpretation of GPL requirements. Official sources available from www.freedesktop.org)

  • gtk+-1.3.0-20030216.zip . Runtime package. This is the gtk-1-3-win32-production branch of GTK+. The GTK+ version is 1.3.0, which is quite old. The GTK+ API had not yet changed much at that time, it is still mostly like GTK+ 1.2.7. Requires GLib 2.0 or 2.2.0.

  • gtk+-dev-1.3.0-20030115.zip. Corresponding developer package.

  • The source for this GTK+ version is in gtk+-1.3.0-20030115-src.zip. There is no other distribution of the source to this GTK+ branch.

  • dirent.zip. The public domain <dirent.h> implementation from mingw, separately packaged for MSVC users. Not directly related to GLib, but here for the convenience of MSVC users, as there used to be a dirent wrapper using #defines in earlier GLib versions for Win32. Needed only by developers that use MSVC.

    In the current GLib, there are real dirent wrappers (see <gdir.h>).

  • gimp-1.2.4-20030213.zip. GIMP 1.2.4 prerelease. GIMP 1.2.4 is not yet released officially. Requires GLib 2.x and GTK+ 1.3.0. Some plug-ins require zlib, libpng, libjpeg, libtiff and freetype. (The only difference between this file and the previous version, gimp-1.2.4-20030115.zip, is a few fixed translations.)

  • gimp-dev-1.2.4-20030115.zip. Corresponding developer package, i.e. libraries and headers for developing and building GIMP plug-ins.

  • gimp-1.2.4-tml-20030119-src.zip. GIMP sources from my source tree. If you redistribute the GIMP executables, you must also redistribute this file. Corresponds to what was in CVS on the date indicated, plus some changes by me for Win32 builds that I haven't committed to CVS, because they are hacks that should be done better, and I don't want to risk disturbing Unix builds. Unless you are very sure that you want to build from exactly the same sources as I happened to do at the date indicated, do consider getting an official source distribution instead.

    In addition to GIMP sources, also the sources to the unofficial plug-ins included in the GIMP zipfile.

  • Zlib, libpng, libjpeg, libtiff and FreeType2. As ported and packaged for Windows by the GnuWin32 project. These are used both by GIMP, and by GTK+ 2.2.x below (except FreeType2, which is used just by GIMP's freetype plug-in and Pango's PangoFT2 engine). The "bin" packages are the runtime ones, the "lib" packages are the ones needed by developers.

  • gimp-gif-1.2.3-20020310.zip. The gif plug-in. Covered by the Unisys LZW patent, so don't download this if you live in a country where the patent is valid and you don't have a license!

  • xpm-nox-4.0.3-20020617.zip A version of the Xpm library that does not use X. Use by the xpm plug-in in GIMP.

  • xpm-nox-dev-4.0.3-20020112.zip, corresponding developer package.

  • gimp-pspi-1.0.2-src.zip. Sources for the pspi plug-in.

The packages below are GTK+ 2.2.x and its dependents. GIMP 1.2.x does not use these.

  • gtk+-2.2.1.zip GTK+ 2.2.1. Requires glib, atk, pango, zlib, libpng, libjpeg and libtiff.

  • gtk+-dev-2.2.1.zip, corresponding developer package.

  • gtk+-2.2.1.tar.gz, corresponding sources.

  • atk-1.0.3-20020821.zip, pango-1.2.1.zip. Used by GTK+ 2.2.1.

  • atk-dev-1.0.3-20020821.zip, pango-dev-1.2.1.zip. Corresponding developer packages.

  • atk-1.0.3.tar.gz, pango-1.2.1.tar.gz Corresponding sources.

  • fontconfig-1.0.1.zip. Used by pangoft2, the FreeType2 backend in Pango. GTK+ does not use pangoft2, so you probably will not need this.

    Fontconfig uses the expat library. Download expat.zip and put the xmlparse.dll in your PATH.
